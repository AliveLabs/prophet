Integration Steps

Load Fraunces font in layout.tsx

Add Fraunces via next/font/google with weights 300–700, assign to --font-fraunces. Add the variable class to <body>.

Import the theme CSS

Import neat-vatic-theme.css after globals.css in your app entry. The [data-brand="neat"] selector means it only activates when the attribute is present.

Set the brand attribute

Add data-brand="neat" to the <html> element. This can be conditional (e.g., based on subdomain, org type, or feature flag) so the same codebase serves both Vatic-generic and Neat-branded experiences.

Verify dark mode

The theme handles .dark class combinations. Ensure next-themes applies .dark to <html> as usual — the theme's dark selectors will cascade correctly.

Layout.tsx Font Setup

// Add alongside existing font imports import { Fraunces } from 'next/font/google'; const fraunces = Fraunces({ subsets: ['latin'], weight: ['300', '400', '500', '700'], style: ['normal', 'italic'], variable: '--font-fraunces', }); // Add to body className: <body className={`${inter.variable} ${spaceGrotesk.variable} ${spaceMono.variable} ${fraunces.variable}`}>

Architecture Note — Vertical Theming Pattern

This is the first vertical theme on the Vatic codebase. The pattern — [data-brand="neat"] scoping all overrides — is designed to scale. Future verticals (Ticket, etc.) follow the same structure: one CSS file per brand, activated by a single attribute. No JS-level branching, no build-time splitting. The base Forge tokens in globals.css remain the default; brand layers are additive overrides only.

Neat adds 7 tokens not in the base Vatic system (--neat-copper, --neat-smoke, etc.) for its "yours vs. theirs" data story. These are namespaced to avoid collision and are only referenced by Neat-branded components.
