Ticket uses a three-layer type system where Vatic has two. Instrument Serif handles display/headlines, Barlow Condensed handles wordmark/nav/labels, and Inter stays as the body/UI thread shared across all Vatic verticals. Font overrides: --font-display: var(--font-instrument-serif) + --font-condensed: var(--font-barlow-condensed)

Type Rules
  • Instrument Serif: minimum 1.2rem / 19px on screen
  • Instrument Serif italic: editorial emphasis only
  • Instrument Serif headlines: <60 characters
  • Barlow Condensed weights: 500 (labels/nav) + 600 (headers/wordmark/buttons)
  • Never set Barlow Condensed and Instrument Serif at the same size

Integration Steps

Load Barlow Condensed + Instrument Serif in layout.tsx

Add both via next/font/google. Assign Barlow Condensed to --font-barlow-condensed and Instrument Serif to --font-instrument-serif. Add both variable classes to <body>.

Import the theme CSS

Import ticket-vatic-theme.css after globals.css in your app entry. The [data-brand="ticket"] selector means it only activates when the attribute is present.

Set the brand attribute

Add data-brand="ticket" to the <html> element. This can be conditional (e.g., based on subdomain, org type, or feature flag) so the same codebase serves both Vatic-generic and Ticket-branded experiences.

Verify dark mode

The theme handles .dark class combinations. Ensure next-themes applies .dark to <html> as usual — the theme's dark selectors will cascade correctly.

Layout.tsx Font Setup

// Add alongside existing font imports import { Barlow_Condensed, Instrument_Serif } from 'next/font/google'; const barlowCondensed = Barlow_Condensed({ subsets: ['latin'], weight: ['500', '600'], variable: '--font-barlow-condensed', }); const instrumentSerif = Instrument_Serif({ subsets: ['latin'], weight: ['400'], style: ['normal', 'italic'], variable: '--font-instrument-serif', }); // Add to body className: <body className={`${inter.variable} ${spaceGrotesk.variable} ${spaceMono.variable} ${barlowCondensed.variable} ${instrumentSerif.variable}`}>

Architecture Note — Vertical Theming Pattern

Ticket is the second vertical theme on the Vatic codebase, following Neat. The pattern — [data-brand="ticket"] scoping all overrides — is identical. One CSS file per brand, activated by a single attribute. No JS-level branching, no build-time splitting. The base Forge tokens in globals.css remain the default; brand layers are additive overrides only.

Ticket adds 10 namespaced tokens not in the base Vatic system (--ticket-rust, --ticket-slate, --ticket-signal, etc.) for its print/service vocabulary. Ticket also introduces a third font layer (Barlow Condensed) that Neat doesn't have — Neat uses only one display font (Fraunces) whereas Ticket splits display and condensed/structural duties across two families.

Key Differences from Neat
  • Primary CTA: Slate (cool blue-gray) instead of Barrel Brown — gives Ticket a cooler, more institutional feel
  • Accent: Rust instead of Proof Copper — warmer and more aggressive
  • Typography: Three-layer system (Instrument Serif + Barlow Condensed + Inter) vs. Neat's two-layer (Fraunces + Inter)
  • Token vocabulary: Print/service ("Ink", "Press", "Counter", "Thermal") vs. Neat's distillery ("Parchment", "Char", "Oak")
  • Dark surfaces: Ink/Press/Counter elevation vs. Neat's Charred Oak/Barrel Dark/Stave
